# Asya Voice Assistant - Документация для ИИ-агентов

> **Текущий статус (MVP):** реализована только часть Phase 1 — wake-word (Vosk) на колонке
> + STT (faster-whisper) на сервере, всё локально на одной машине, без TTS/NLU/БД.
> Код лежит в `speaker/main.py` и `server/server.py` (а не в корне репозитория).

## Обзор проекта

**Asya** — локальный голосовой ассистент, похожий на Алису или Алексу. Состоит из двух основных компонентов: умной колонки (устройство с микрофоном) и серверной части (обработка, логика, база данных).

**Ключевые принципы:**
- Полностью локальное — всё работает на домашней сети
- Расширяемая архитектура команд
- ИИ-агент для понимания и ответов на запросы
- История запросов для будущей памяти агента
- В будущем: интеграция с музыкой, умным домом, Телеграм-ботом

## Компоненты системы

### 1. Умная колонка (Speaker/Device)

**Текущее состояние:** `main.py` (основной цикл обработки аудио)

**Функции:**
- Непрерывное прослушивание аудиопотока
- Детекция пробуждающего слова "ася" с помощью Vosk
- Запись аудио после активации (до достижения тишины)
- Отправка записанного аудио на сервер по WebSocket
- Воспроизведение ответа от сервера через динамик
- Светодиодный индикатор статуса (LED на GPIO 17)

**Параметры конфигурации:**
```python
SAMPLE_RATE = 16000  # Hz
FRAME_MS = 30
WAKE_WORD = "ася"
SILENCE_TIMEOUT_MS = 1500  # Время тишины для завершения записи
LED_PIN = 17
SERVER_URL = "ws://192.168.1.100:8000/ws"  # WebSocket endpoint
```

**Требования при разработке:**
- Поддерживать WebSocket соединение с сервером
- Обработать случаи разрыва соединения и переподключения
- Отправлять WAV аудиофайлы или аудиопоток
- Получать и воспроизводить аудиоответ

### 2. Серверная часть

**Требуемые компоненты:**

#### 2.1 API (FastAPI рекомендуется)
- **WebSocket endpoint** `/ws` — основной канал коммуникации
  - Получение аудиопотока от колонки
  - Отправка ответа обратно
  - Управление состоянием соединения
  
- **HTTP endpoints:**
  - `POST /api/audio` — загрузка аудиофайла (для совместимости)
  - `GET /api/commands` — список доступных команд
  - `POST /api/commands` — добавление новой команды
  - `GET /api/history` — история запросов
  - `GET /api/health` — проверка здоровья сервера

#### 2.2 Аудиообработка (Audio Processing Pipeline)

```
WAV файл → STT (Speech-to-Text) → Текст запроса
                                      ↓
                              [AI Agent processing]
                                      ↓
                    Текст ответа → TTS (Text-to-Speech) → WAV файл
```

**STT (Speech-to-Text):**
- Текущее решение: Vosk для локальной обработки
- Альтернативы: Whisper (OpenAI), другие локальные модели
- Возвращает текст команды

**TTS (Text-to-Speech):**
- Требование: локальный движок
- Варианты:
  - Silero TTS (русский язык, хорошее качество)
  - pyttsx3 (кроссплатформенный)
  - Coqui TTS
- Возвращает WAV аудиофайл

#### 2.3 AI Agent (Интеллектуальное ядро)

**Ответственность:**
- **NLU (Natural Language Understanding):** понимание намерения пользователя
- **Command Routing:** маршрутизация на нужный обработчик
- **Context Awareness:** использование истории для лучшего понимания
- **Response Generation:** формирование ответа на естественном языке

**Входные данные:**
- Текст распознанной команды
- История запросов (последние N запросов)
- Доступные команды (список функций)
- Текущее время, дата

**Выходные данные:**
- Текст ответа (для TTS)
- Команда для выполнения (если требуется)
- Параметры команды
- Уверенность в понимании запроса

**Примеры команд:**
```
User: "Ася, который час?"
→ Command: TIME
→ Response: "Сейчас 14 часов 30 минут"

User: "Ася, какая погода?"
→ Command: WEATHER
→ Response: "Сегодня солнечно, температура 22 градуса"

User: "Ася, включи свет в гостиной"
→ Command: SMART_HOME
  params: {device: "light", room: "living_room", action: "on"}
→ Response: "Включил свет в гостиной"
```

#### 2.4 Command System (Система команд)

**Архитектура:** плагины/обработчики для каждой команды

```
CommandRouter
├─ BuiltInCommands
│  ├─ TIME (текущее время)
│  ├─ DATE (текущая дата)
│  ├─ WEATHER (погода - требует API)
│  ├─ HELP (справка по командам)
│  └─ ...
├─ SmartHomeCommands (future)
│  ├─ LIGHT (управление освещением)
│  ├─ TEMPERATURE (управление климатом)
│  └─ ...
└─ MusicCommands (future)
   ├─ PLAY (воспроизведение музыки)
   ├─ STOP (остановка)
   └─ ...
```

**Как добавить новую команду:**

1. Создать обработчик команды
2. Зарегистрировать в CommandRegistry
3. Добавить примеры в AI Agent для NLU
4. Обновить список команд

#### 2.5 База данных

**Хранимые данные:**

```sql
-- Request history (для памяти агента)
requests (
  id INTEGER PRIMARY KEY,
  timestamp DATETIME,
  user_input TEXT,
  ai_response TEXT,
  command_executed TEXT,
  status ENUM(success, error, unknown),
  confidence FLOAT,
  session_id TEXT
)

-- Command log (для аналитики)
commands (
  id INTEGER PRIMARY KEY,
  timestamp DATETIME,
  command_name TEXT,
  parameters JSON,
  result TEXT,
  duration_ms INTEGER
)

-- User preferences (для персонализации)
user_settings (
  key TEXT,
  value TEXT,
  updated_at DATETIME
)

-- Configuration
config (
  key TEXT PRIMARY KEY,
  value TEXT
)
```

**Рекомендуемая БД:** SQLite (простая, локальная) или PostgreSQL (масштабируемость)

## Поток обработки запроса

1. **Активация (на колонке)**
   - Пользователь говорит: "Ася, ..."
   - Vosk детектирует wake word
   - Начинается запись аудио
   - После тишины (1.5s) запись завершается

2. **Отправка (колонка → сервер)**
   - WAV файл отправляется по WebSocket
   - Отправляется уникальный request_id для отслеживания

3. **Обработка на сервере**
   - **STT:** преобразование аудио в текст
   - **NLU:** анализ текста, определение намерения
   - **Action:** выполнение команды (если требуется)
   - **Response:** формирование текстового ответа
   - **TTS:** преобразование ответа в аудио

4. **Отправка ответа (сервер → колонка)**
   - WAV файл с ответом отправляется обратно по WebSocket
   - Параметры параллельной обработки

5. **Воспроизведение (на колонке)**
   - Аудиоответ воспроизводится через динамик
   - LED выключается

6. **Логирование (сервер)**
   - Весь процесс логируется в БД
   - Сохраняется история для будущей памяти агента

```json
{
  "type": "audio_request",
  "request_id": "uuid-12345",
  "timestamp": "2024-01-15T14:30:00Z",
  "device_id": "speaker_01",
  "audio_data": "<base64 encoded WAV>",
  "format": "wav",
```

### Сообщение: Ответ от сервера

```json
{
  "type": "audio_response",
  "request_id": "uuid-12345",
  "status": "success",
  "text_response": "Сейчас 14 часов 30 минут",
  "audio_data": "<base64 encoded WAV>",
  "command_executed": "TIME",
  "confidence": 0.95,
  "metadata": {
    "processing_time_ms": 1200,
    "source": "ai_agent"
  }
}
```

```json
{
  "type": "error",
  "request_id": "uuid-12345",
  "error_code": "STT_ERROR",
  "message": "Не смог распознать речь",
  "fallback_text": "Извините, я не понял. Повторите пожалуйста."
}
```

3. **Локальность:** приоритет локальным решениям над облачными
4. **Надежность:** обработка ошибок и переподключения
5. **Логирование:** всё должно логироваться в БД

### Технический стек (рекомендация)

**Backend:**
- FastAPI (Python) — быстро, асинхронно, WebSocket support
- SQLAlchemy — ORM для БД
- SQLite или PostgreSQL — база данных

**Audio Processing:**
- Vosk — локальное распознавание (wake word + STT)
- Silero TTS или pyttsx3 — синтез речи
- librosa или scipy — обработка аудио

**AI/NLU:**
- Claude API (local agent) или локальная LLM (Ollama)
- Простые паттерны и регулярные выражения для базовых команд
- Intent classification для более сложных запросов

**Deployment:**
- Docker контейнер для сервера
- systemd или supervisord для управления процессами
- Локальная сеть (192.168.x.x)

```python
class TimeCommand:
    name = "TIME"
    description = "Сказать текущее время"
    
    def execute(self):
        from datetime import datetime
        now = datetime.now()
        return f"Сейчас {now.strftime('%H часов %M минут')}"
    
    def can_handle(self, text: str) -> bool:
        patterns = [
            r"который час",
            r"во сколько",
            r"текущее время",
            r"скажи время"
        ]
        return any(re.search(p, text.lower()) for p in patterns)
```

## Обработка ошибок

**Сценарии, которые должны обрабатываться:**

1. **STT не смог распознать** → "Извините, я не понял. Повторите пожалуйста."
2. **Команда неизвестна** → "Такую команду я еще не поддерживаю"
3. **Внешний API недоступен (погода)** → "Не смог получить информацию о погоде"
4. **WebSocket разорван** → автоматическое переподключение на колонке
5. **БД недоступна** → логирование в файл, возобновление при восстановлении

## Мониторинг и отладка

- `GET /api/health` — здоровье системы
